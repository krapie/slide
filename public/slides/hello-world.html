02 / 05

What it does.

01

List & Browse

All slides are indexed in a manifest and presented as a browsable card grid at the root of the service.

02

Full-screen Viewer

Each slide opens in a full-viewport iframe — no chrome, no distractions. Direct link opens raw HTML.

03

Code-first

Each slide is a standalone HTML/CSS file generated from a slash command and committed directly to the repository.

04

Git-deployed

Commit the generated file, push to main. GitHub Actions builds and ArgoCD syncs the cluster within minutes.

03 / 05

How it works.

01

Run the command

Open the slide repo and run the slash command to scaffold a new slide.

/new-slide My Talk

02

File generated

A self-contained HTML/CSS file is created and the manifest index is updated.

public/slides/my-talk.html

03

Commit & push

Push to main. CI builds the Docker image and updates the deployment tag.

git push origin main

04

Live in ~3 min

ArgoCD detects the new image tag and syncs the cluster automatically.

slide.kevinprk.com

Stack: Vite + React (list/viewer) · Nginx (static) · k8s + ArgoCD (deploy) · GitHub Actions (CI)
